# Лендинг «Собери сайт дома» — 2 кейса

Готовый лендинг с двумя табами:

- **Поворот головы** — портрет в CRT-шлеме, голова следует за курсором (кейс «kernel»).
- **Светлая / тёмная тема** — кинематографичный кемпинг день/ночь (кейс «Emberline»).

Каждый таб даёт клиенту всё, чтобы собрать сайт у себя:
- **архив** (промпт пересборки + видео) — рядом с названием;
- раздел **«Как собрать сайт»** — шаги + скачивание промпта и вложений;
- раздел **«Промпты для генерации фото и видео»** — тексты промптов с кнопкой «Копировать» и все вложения/результаты на скачивание.

## Как это устроено (важно)

Сам лендинг — это только **4 лёгких файла** (`index.html`, `styles.css`, `app.js`, `data.js`).
Все тяжёлые файлы (видео, фото, архивы) **не лежат в сайте** — они хостятся в
**GitHub Releases**. Так страница грузится мгновенно, а ссылки на скачивание отдаёт GitHub.

Все ссылки собираются из одной константы вверху `data.js`:

```js
const REPO = "MatveiShulga/dop-materiali-sites";
const TAG  = "v1";
```

## Запуск за 2 шага

1. **Залить медиа в GitHub Releases** (один раз):
   ```bash
   gh auth login              # если ещё не залогинен
   bash upload-to-github.sh   # создаст релиз v1 и зальёт всё из release-assets/
   ```
   > Хочешь другой репозиторий/тег — поменяй `REPO`/`TAG` **и в `data.js`, и в `upload-to-github.sh`** (значения должны совпадать).

2. **Выложить сам лендинг** — залей `index.html`, `styles.css`, `app.js`, `data.js`
   на любой статический хостинг (GitHub Pages, Netlify, Vercel, Cloudflare Pages).
   Папку `release-assets/` и скрипты выкладывать **не нужно** — это только для заливки в Releases.

## Что внутри `release-assets/`

| `site1-photo-main / left / right.jpg` | Портрет: основное фото и повороты |
| `site1-ref.webp` | Референс-вложение для первого промпта |
| `site1-video.mp4` | Hero-видео «поворот головы» |
| `site1-recreate-prompt.md` | Промпт пересборки сайта 1 |
| `site1-archive.zip` | Промпт + видео (набор для сборки) |
| `site2-photo-day / night.jpg` | Кемпинг: день и ночь |
| `site2-video-day / night.mp4` | Видео-фоны день/ночь |
| `site2-recreate-prompt.md` | Промпт пересборки сайта 2 |
| `site2-archive.zip` | Промпт + оба видео (набор для сборки) |

## Локальный предпросмотр

Ссылки на медиа абсолютные (GitHub), поэтому после заливки можно просто открыть
`index.html` в браузере — фото/видео подтянутся с GitHub. До заливки превью-картинки
будут «битыми» — это нормально.
